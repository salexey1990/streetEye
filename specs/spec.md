# Спецификация проекта StreetEye

## 📱 Общая информация

**Название:** StreetEye  
**Слоган:** «Видь город иначе»  
**Тип:** Мобильное приложение-тренажёр для стрит-фотографов  
**Платформы:** iOS, Android (веб-версия опционально)  
**Языки:** Русский, английский (с возможностью расширения)

---

## 🎯 Цель проекта

Помочь фотографу выйти из зоны комфорта, перестать снимать «одно и то же» и научиться видеть город по-новому через систему структурированных заданий и челленджей.

### Основные задачи
- Борьба с творческим выгоранием
- Дисциплина и регулярная практика
- Обучение через действие
- Развитие насмотренности и креативности

---

## 👥 Целевая аудитория

| Сегмент | Описание | Потребности |
|---------|----------|-------------|
| **Новички** | Начинающие стрит-фотографы | Базовые техники, уверенность, простые задания |
| **Любители** | Фотографы с опытом 1-3 года | Разнообразие, преодоление рутины, рост навыков |
| **Профи** | Опытные фотографы | Хардкор-челленджи, технические ограничения, портфолио |

---

## 🛠 Ключевые функции

### 1. Умная генерация заданий

#### Категории заданий:
| Категория | Описание | Примеры |
|-----------|----------|---------|
| **🎯 Технические** | Работа с камерой и настройками | «Только 35мм», «Ручная фокусировка», «Длинная выдержка» |
| **🎨 Визуальные** | Композиция, свет, цвет | «Тени», «Отражения», «Геометрия», «Неон» |
| **👥 Социальные/Эмоциональные** | Люди, взаимодействия, истории | «Одиночество в толпе», «Руки», «Взгляд в камеру» |
| **🚫 Ограничения** | Искусственные ограничения | «Без людей», «Только ч/б», «Лимит 10 кадров» |

#### Уровни сложности:
- **Новичок** — базовые техники, простые концепции
- **Средний** — комбинированные задания, требующие навыков
- **Профи** — сложные ограничения, творческие вызовы

**Контент:** 80+ заданий (по 20 на категорию), распределённых по 3 уровням сложности.

---

### 2. Режимы работы

| Режим | Описание | Длительность |
|-------|----------|--------------|
| **🚶 Quick Walk** | Одно случайное задание на прогулку | 30–60 минут |
| **🔥 Heat Mode** | Новое задание каждые 15 минут | 1–2 часа |
| **📅 7-Day Challenge** | Серия связанных заданий на неделю | 7 дней |
| **📍 Location Based** | Задания на основе геолокации | По ситуации |

---

### 3. Недельные марафоны

Готовые сценарии на 7 дней с прогрессией сложности:

| Марафон              | Цель                            | Итог                        |
| -------------------- | ------------------------------- | --------------------------- |
| **Композиция**       | Осознанное построение кадра     | 7 кадров с разными приёмами |
| **Свет**             | Работа с разным освещением      | Серия «24 часа света»       |
| **Человек в городе** | Преодоление страха съёмки людей | Эмоциональная серия         |
| **Минимализм**       | Умение видеть суть              | «Тихая» галерея             |
| **Техника**          | Полный ручной контроль          | Прогресс от авто к мануалу  |
|                      |                                 |                             |

---

### 4. Обучающий блок (Tips & Tricks)

К каждому заданию прилагается:
- **Краткая справка** — техника выполнения
- **Совет** — практические рекомендации
- **Примеры** — работы известных фотографов (Фан Хо, Вивиан Майер и др.)
- **Мини-тест** — проверка понимания (для технических заданий)

---

### 5. Дневник прогресса (Private Journal)

- Прикрепление лучшего кадра к заданию (локальное хранение)
- Заметки: «Что было сложно?», «Что получилось?»
- Статистика по выполненным заданиям
- Трекер достижений (бейджи за серии заданий)
- Автоматический коллаж по итогам марафона (2×3 или 3×3)

---

### 6. AI-анализ (Продвинутая функция)

- Загрузка фото по заданию
- Анализ соответствия заданию
- Обратная связь с рекомендациями
- Персонализированное следующее задание на основе слабых мест

---

## 🎨 Интерфейс и UX

### Принципы дизайна
- **Dark Mode по умолчанию** — не привлекает внимание на улице
- **Крупные элементы** — управление одной рукой
- **Минимализм** — ничего лишнего, фокус на задании
- **Быстрый доступ** — виджет на главный экран

### Специальные режимы
| Режим | Описание |
|-------|----------|
| **Инкогнито** | Интерфейс как заметки/текст (от взгляда через плечо) |
| **Watch Integration** | Вибрация на часах для смены задания (Apple Watch / Android Wear) |
| **Оффлайн-режим** | Работа без интернета (задания кэшируются) |

## 📊 Структура контента

### База заданий (MVP: 100+)

```
Задание:
├── Название
├── Категория (Техническое / Визуальное / Социальное / Ограничение)
├── Уровень сложности (Новичок / Средний / Профи)
├── Описание
├── Подсказка (Tips & Tricks)
├── Примеры (опционально, изображения)
├── Теги (для фильтрации)
└── Связанные марафоны (если применимо)
```

### Марафоны (5 готовых сценариев)

```
Марафон:
├── Название
├── Описание цели
├── Дней: 7
├── Задания по дням [1-7]
│   ├── День
│   ├── Задание (ссылка на базу)
│   ├── Уровень
│   └── Связь с предыдущим днём
└── Итоговый бонус (коллаж, бейдж)
```

## 🏗 Техническая архитектура

### Стек технологий

| Компонент | Технология | Обоснование |
|-----------|------------|-------------|
| **Мобильное приложение** | React Native | Кроссплатформенность (iOS/Android), единая кодовая база, большая экосистема |
| **Бэкенд** | NestJS | Модульная архитектура, TypeScript, поддержка микросервисов из коробки |
| **Архитектура бэкенда** | Микросервисы в монорепозитории | Изоляция доменов, независимое масштабирование, удобство разработки |
| **База данных** | PostgreSQL | Надёжность, ACID, сложные запросы, JSONB для гибких схем |
| **Кэш** | Redis | Сессии,.rate limiting, кэширование частых запросов |
| **Хранилище фото** | S3-совместимое (MinIO / AWS S3) | Надёжность, CDN, версионирование |
| **Очереди сообщений** | RabbitMQ / Kafka | Асинхронная коммуникация между микросервисами |
| **AI-анализ** | Python-сервис (FastAPI) + TensorFlow/PyTorch | Специализированный сервис для ML-задач |
| **Push-уведомления** | Firebase Cloud Messaging + APNs | Кроссплатформенные уведомления |
| **Геолокация** | Native GPS API + PostGIS | Location-based задания, гео-запросы |
| **Контейнеризация** | Docker + Docker Compose | Изоляция сервисов, воспроизводимость окружения |
| **Оркестрация** | Kubernetes (опционально для продакшена) | Масштабирование, отказоустойчивость |

## 📝 Заключение

**StreetEye** заполняет пробел на рынке между соревновательными платформами и образовательными курсами. Приложение предлагает уникальный подход: **обучение через практику** с персонализированными заданиями, структурированными марафонами и приватным прогресс-трекингом.

**MVP готов к запуску за 6 недель** с базой из 100 заданий, рандомайзером и дневником прогресса. Дальнейшие этапы добавят премиум-функции, AI-анализ и интеграцию с носимыми устройствами.
